🔺一个类的描述

  • ② 属性 —— 静态描述类的特征（变量 存值） name
  • ① 方法 —— 动态描述类的行为（做事情）eat
  • ③ 构造方法 —— 用来创建当前类的对象
  • ④ 程序块(代码块) —— 理解为就是一个方法（非常特殊：无参数 无修饰符 无返回值 无名字）
  • （ ⑤ this关键字的作用 —— 用来代替某一个对象
  • （⑥ 类的加载对象的创建（内存原理 机制）

类中四成员

1. 属性

  1. 静态描述特征（存值）
  2. 值若不写，有默认值

2. 方法

  1. 动态描述行为（做事情）

  2. 最主要的是 方法设计的参数和返回值问题 —— 传递 调用 执行 内存

  3. 方法重载overload (一个类中，一组方法：名字相同 参数列表不同(个数/类型/顺序))

  4. 动态参数列表

3. 构造方法

  1. 唯一作用——创建当前类对象（做事情）
  2. 结构和写法：无返回类型，有返回值
  3. 调用：通过new关键字
  4. 特点：每一个类都有默认无参数的构造方法，我们自定义了新的，默认的构造方法即被覆盖

4. 程序块(代码块)

  1. 可以理解为一个特殊的方法
  2. （没有修饰符，没有返回类型，没有名字，没有参数）只有一个方法的执行体 {}
  3. 程序块是每一次调用构造方法之前都会默认，自动执行（无需调用 构建对象前调用）
  4. 程序块没有重载，却可以存在多个，按照创建的顺序逐一执行

  2. this既然是一个对象，可以调用一般的属性，可以调用一般的方法

程序块(代码块) { }

1. 作用

跟普通方法一样，做事情的

2. 写法

可以认为程序块是一个没有修饰符 没有返回值 没有参数 没有名字 的特殊方法

3. 用法

块也需要调用才能执行，我们自己调用不到（因为没有名字）

每一次我们调用构造方法之前，系统会帮我们自动调用一次程序，让她执行一遍

4. 特点

  • 无重载的概念（因为无名字 无参数）
  • 但可以在类中定义 多个程序块
  • 可以在里面写一些程序，在创建对象之前执行

nextLine()

方法会以回车符为截止，读取回车符+之前的所有字符，将回车符扔掉，把之前所有字符组合成一个完整的字符串，交换给我们 【只有nextLine读取回车符】

nextInt() （nextFloat、nextDouble、nextByte）

方法会以回车符为截止，读取回车符之前所有字符，回车符留在队列中

【消息队列】
nextLine() -> nextInt() 游 蜜 桃 回车 1 2 3 回车
nextInt() -> nextLine() 1 2 3 回车 ①
next() a b c 空格 1 2 3 空格 e q
nextLine() a b c 空格 1 2 3 空格 e q e ①

①nextLine读取到了回车，保存结果为""，不再等待下一步的用户输入

① scannerTest.java ——==正常 √== 阻塞效果，等待输入

② scannerTest.java —— ==不正常 ×== 未阻塞，还没输入就读取到东西了

③ 解决方案

  1. 补充一行nextLine，将nextInt留下的回车符回收，让下一个nextLine正常读取

  2. next()，从前面的回车之后才开始读取
